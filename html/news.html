2013

  • August:
    • The Registry now provides a dataset descriptor. This describes the content of the Registry as a whole, using guidelines agreed upon by numerous communities, including HCLS, OpenPHACTS, Bio2RDF and BioSharing/BioDBCore. It makes use of terms from the following vocabularies: DCMI, VoID, PAV, FOAF and VOAG. The dataset descriptor can be accessed from the Registry's download page and is available in RDF/XML and Turtle formats.
    • Metadata for data collections in the Registry is now provided in RDF. This representation uses DCAT, an RDF vocabulary specifically designed to describe different datasets (data collections) in data catalogs, such as the Registry. A RDF export of the entire content of the Registry is available in RDF/XML and Turtle formats. In addition, an RDF representation of individual data collection can be downloaded from each data collection's page (via the green tabs on the top right).
  • July:
    • As requested by a number of our users, Identifiers.org and its Registry now highlight the "primary resource" for data collections. This represents the resource that is maintained by the provider of the original data, and thereby acknowledges the original source. Please note that not all data collections have a primary resource; for example where multiple collaborating institutes are equally responsible, no primary resource is specified.

2011

  • December:
    • The XML export and XML Schema have been updated. For each data collection, the XML export now includes various new elements such as the namespace, the root Identifiers.org URL or the list of restrictions (if any recorded).
    • Introduction of a mechanism to state specific restrictions associated with certain data collections. These restrictions include restricted licenses (which for example may preclude the use of a data collection by commercial organisations) and non-stable identifiers (which for example may differ for a specific record between releases or versions of a data collection). The introduction of this system will enable us to publish data collections that were previously not compliant with the Registry's requirements. However, although this will ensure provision of URIs for records provided by those data collections, the level of support and services (such as the resolution one) may be affected. We welcome feedback on this recent development, and suggestions for its future refinement.
    • Identifiers.org and MIRIAM Registry: community resources to provide persistent identification published. In this paper, we describe the MIRIAM Registry, which provides unique, perennial and location-independent identifiers for data used in the biomedical domain. We also introduce the new Identifiers.org service that is built upon the information stored in the Registry and which provides directly resolvable identifiers, in the form of Uniform Resource Locators (URLs). The flexibility of the identification scheme and resolving system allows its use in many different fields, where unambiguous and perennial identification of data entities are necessary.

2010

  • April:
    • Web Services updated: getURI() and getLocations() now perform a check of the validity of the identifier provided. In case it is not valid, the result returned is null. For more information about these methods, please refer to the Web Services documentation.
    • The Java library is now available via the EBI Maven repository. Cf. the library page for more details.
  • March:
    • On the detail page of a data collection, the deprecated resources are not displayed by default.
    • Access to the data is now done via shorter and nicer URLs:
      • Access to a data collection: @MIR_DYNAMIC_URL@/datatypes/MIR:00000008
      • Access to a resource: @MIR_DYNAMIC_URL@/resources/MIR:00100005
      • Access to the tag(s) associated to a data collection: @MIR_DYNAMIC_URL@/tags/MIR:00000005
      • Access to the data collections associated with a tag: @MIR_DYNAMIC_URL@/tags/MIR:00600006
      • Access to examples of usage of a data collection: @MIR_DYNAMIC_URL@/usage/MIR:00000004
      • Access to the data collections which can be used to annotate a given element: @MIR_DYNAMIC_URL@/usage/MIR:00500003
      • Access to the Web Services provided by a data collection: @MIR_DYNAMIC_URL@/webservices/MIR:00000008
    • Information about Web Services provided by the various data collections is now available via a fourth tab. These records will be gradually completed.
    • We started to perform a daily check of the health of all the resources several months ago, for maintenance purposes. Now this information is available to everybody. You just need to click on a resource identifier to access its health history. Moreover, the resources are displayed ordered by reliability.
    • The BioModels.net qualifiers are now available to download as an XML file and RDF file. We welcome any suggestions and comments to improve these. BETA
    • A new version of the XML export is now available, with direct access to the file XML file. This file is generated on demand, so you are sure to always get an up-to-date export.
    • REST Web Services are now available from http://www.ebi.ac.uk/miriamws/main/rest/ for you to test and give us feedback! BETA
    • New version of the Java library now available (download from SourceForge).
    • SOAP Web Services updated: various new kind of parameters now accepted by the method getOfficialDataTypeURI(). Cf. the page describing the available services for more information.
  • January:
    • More than 100 data collections now in MIRIAM Resources!
